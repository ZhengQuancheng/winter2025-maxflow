## 贸易网络最大流量规划（CUDA）

### **任务背景**

“天下熙熙皆为利来，天下攘攘皆为利往”。泛星系贸易市场的成立使得无数星球之间通过超空间航道连接，每条航道有固定的货运容量（万吨/天）。凭借你 InfiniTensor 训练营的经历和人工智能与并行计算方面的出色成绩，你被晋升为泛星系贸易管理局的算法官，负责实时计算从资源富集星球（源）到需求中心（汇）的最大货物运输量，以规划贸易路线、避免拥堵。

泛星系贸易市场体量巨大，星球数量已达百万，航道数量千万，传统 CPU 无法满足实时查询需求。你需要开发并行最大流计算程序，以闪电般的速度响应各星区的物流请求。如果效果显著，你的程序将被部署于国家级 “星枢” 计算阵列，直接影响国家的贸易，甚至决定一场星际饥荒的存亡。

### **领域知识简介**

有向图（Directed Graph）是一种图结构，其中的边具有方向性。这意味着每条边都有一个起始节点和一个结束节点，表示数据/状态/进程从一个节点流向另一个节点。给定一个有向图，每条边有容量，求从源节点 `s` 到汇节点 `t` 能输送的最大流量。你需要 “设计” 一个并行友好的算法。

### **任务内容**

开发一个 CUDA 程序，实现并行最大流计算，处理单源单汇最大流，并输出规定的输出文件。

#### **输入文件定义**

1. **图文件（CSR 格式二进制）**：CSR（Compressed Sparse Row）格式存储有向图，包含三个数组：

- `row_ptr`：长度为 N+1 的整数数组，`row_ptr[i]` 表示节点 i 的出边在 `col_indices` 和 `values` 中的起始索引，`row_ptr[N]` 为总边数 M。
- `col_indices`：长度为 M 的整数数组，存储每条边的目标节点。
- `values`：长度为 M 的浮点数或整数数组，存储每条边的容量。

示例：一个简单图：节点 0→1(容量 5)，节点 0→2(容量 3)，节点 1→2(容量 2)。则：

```
row_ptr = [0, 2, 3, 3]    // 节点0有2条边，节点1有1条，节点2有0条
col_indices = [1, 2, 2]   // 节点0的出边目标：1和2；节点1的出边目标：2
values = [5.0, 3.0, 2.0]  // 对应容量
```

文件存储为二进制，顺序：先写入 N（int），M（int），然后写入 `row_ptr` 数组（N+1 个 int），`col_indices`（M 个 int），`values`（M 个 float）。

1. **源汇查询文件（文本格式）**包含多个查询对，每行分别是：`source_id target_id`。

```
0 2
1 2
0 1
```

#### 输出文件定义

1. **最大流结果文件**：对于每个查询，输出一行 `source target maxflow`。
2. **性能日志**：总查询时间，平均每查询时间（TPQ），首次查询时间（TTFQ），以及吞吐量（边更新次数/秒）等。

### 要求

1. **图规模**：图文件采用 CSR 格式，程序需支持节点数至少 `10^5`，边数至少 `10^6`。初期测试可使用小图。
2. **数据类型**：统一为 `float` 或 `int`。
3. **多查询支持**：支持至少 10 个不同的源汇对查询，可串行处理，但需统计总查询时间和平均每查询时间（TPQ）。
4. **正确性验证**：与 CPU 串行实现在小图上对比，确保最大流值一致。
5. **性能指标**：输出算法运行时间、吞吐量（边更新次数/秒）、收敛迭代次数。
6. **内存管理**：图数据需常驻显存，每次查询仅重新初始化源汇相关数据。
7. **平台适配**：默认需在英伟达平台上支持。每在一款国产平台上支持，额外加 20% 乘算系数。
